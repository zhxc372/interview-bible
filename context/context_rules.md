# Context Rules

## 1. 最小上下文原则

每个生成任务只能读取完成当前任务所需的最小上下文。

## 2. 原始输入只进一次

原始 JD 只允许用于 JD Intake 阶段。
Topic Card 生成阶段禁止重新注入完整 JD。

## 3. 单知识点生成原则

每次只生成一个知识点的卡片。
禁止一次生成多个 P0 知识点的完整内容。

## 4. 项目证据按需注入

项目经历必须先压缩为 project_evidence_index。
生成卡片时只注入与当前 topic 相关的项目证据。

## 5. Markdown 由脚本拼接

最终 Markdown 不由大模型一次性生成。
大模型只负责生成结构化卡片内容。
拼接用 Python 脚本完成。

## 6. P2 降级原则

P2 知识点默认只生成速记版，不生成完整面试卡，除非用户指定。

## 7. 状态持久化原则

每个 session 的进度必须写入 state.yaml。
每次生成卡片后更新 state.yaml。
中断恢复时读取 state.yaml 确定断点。
